# 서비스 배포 전 점검사항 및 해결방법

> 캡처 기준: 환경변수는 로드됨(✓). 단, 배포 시 추가 조치 필요.

---

## 1. 현재 연결 상태 요약

| 항목 | 상태 | 비고 |
|------|------|------|
| 팝빌 API 키 (POPBILL_LINKID, SECRET_KEY) | ✅ 로드됨 | check-env에서 확인 |
| Gemini API 키 | ✅ 설정됨 | 음성/텍스트 분석 |
| AUTH_SECRET | ⚠️ 기본값 사용 | 운영 시 반드시 변경 |
| 사용자 데이터 (users.json) | ⚠️ 로컬 파일 | 배포 시 DB 필요 |

---

## 2. 배포 시 반드시 해결해야 할 항목

### 2-1. AUTH_SECRET 미설정 (보안 위험)

**문제**: JWT 서명에 기본값 사용 시 토큰 위·변조 가능

**해결**:
```bash
# 강력한 시크릿 생성 (PowerShell)
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Maximum 256 }) -as [byte[]])
```
`.env` 또는 배포 플랫폼 환경변수에 추가:
```
AUTH_SECRET=생성된_32바이트_이상_문자열
```

---

### 2-2. 사용자 데이터 저장소 (data/users.json)

**문제**: `/data` 폴더가 로컬 파일시스템에만 존재. Vercel 등 서버리스에서는 **쓰기 불가**.

**해결** (택 1):
- **옵션 A**: Vercel Postgres, Supabase, PlanetScale 등 **DB 연동**
- **옵션 B**: Vercel KV, Upstash Redis 등 **키-값 저장소**
- **옵션 C**: 파일 기반이 필요하면 Railway, Render 등 **영구 디스크 지원 플랫폼** 사용

---

### 2-3. .env 파일은 배포되지 않음

**문제**: `.env*`는 `.gitignore`에 있어 Git에 포함되지 않음. 배포 시 환경변수 없음.

**해결**: 배포 플랫폼에서 환경변수 직접 설정

| 플랫폼 | 설정 위치 |
|--------|-----------|
| Vercel | Project → Settings → Environment Variables |
| Netlify | Site → Build & deploy → Environment |
| Railway | Project → Variables |

**필수 환경변수 목록**:
```
GEMINI_API_KEY=...
AUTH_SECRET=...          # 운영용 강력한 값
POPBILL_LINKID=...
POPBILL_SECRET_KEY=...
POPBILL_IS_TEST=false    # 실제 국세청 발행 시
```

---

### 2-4. POPBILL_IS_TEST (테스트 vs 운영)

**문제**: `true`이면 테스트 환경만 사용. 실제 국세청 전송 불가.

**해결**: 실제 서비스 시
```
POPBILL_IS_TEST=false
```
그리고 팝빌 운영 포인트 충전 필요.

---

### 2-5. API 키 노출 위험

**문제**: `.env.local`에 API 키가 있음. 실수로 커밋되면 노출.

**확인**:
```bash
git status
# .env, .env.local이 목록에 있으면 안 됨
```

**해결**: `.gitignore`에 `.env*` 포함 확인 (현재 ✅ 포함됨)

## 4. 배포 플랫폼별 참고

### Vercel
- 서버리스: `data/users.json` 쓰기 불가 → **DB 필수**
- Edge/Serverless에서 `process.cwd()` + 파일 쓰기 동작 불가

### Railway / Render
- 영구 디스크 지원 가능 → 파일 기반 유지 시 선택지

### Docker (VPS 등)
- 볼륨 마운트로 `/data` 유지 가능
- 환경변수는 `docker run -e` 또는 `.env` 파일로 전달

## 6. 현재 캡처 관련

캡처에서 **"환경변수는 로드되었습니다"**가 보이면:
- `POPBILL_LINKID`, `POPBILL_SECRET_KEY`는 서버에 정상 로드된 상태입니다.
- 그래도 Step 0 노란 박스가 보인다면:
  1. 페이지 새로고침 (F5)
  2. 로그아웃 후 다시 로그인
  3. 브라우저 캐시 삭제 후 재접속

# 음성 기반 세금계산서 발행 & 맞춤형 보조금 찾기 서비스 개발 계획서

> **문서 기반**: `doc/infor.md` 분석 결과  
> **작성일**: 2025-02-24  
> **프로젝트**: B2B SaaS 웹 서비스 MVP

---

## 1. 프로젝트 개요

### 1.1 서비스 목표
- **핵심 기능**: 사용자가 음성으로 명령하면 AI가 세금계산서 필수 데이터를 추출하고, 사용자 확인 후 국세청으로 발행하는 서비스
- **추가 기능**: 소상공인이 사업장 정보를 입력/음성으로 검색하면 받을 수 있는 정부지원금을 AI가 매칭하여 추천

### 1.2 핵심 가치
- **편의성**: "말만 하면 발행되는" 세금계산서
- **정확성**: Human-in-the-loop으로 환각(할루시네이션) 방지
- **보안**: 인증서·민감정보는 서버/ASP에서만 처리

### 1.3 기술 스택

| 구분 | 기술 |
|------|------|
| Frontend | Next.js (App Router), TypeScript, TailwindCSS, lucide-react |
| Backend | Next.js Route Handlers (API Routes) |
| AI | Google Gemini API (gemini-1.5-flash / Pro) |
| 외부 연동 | Popbill 또는 Barobill Node.js SDK (세금계산서 발행) |
| 추가 데이터 | 공공데이터포털 기업마당 API (보조금 검색) |

## 2. 시스템 아키텍처

### 2.1 전체 흐름도

```
[Client] 음성 녹음 → [Server] /api/analyze-voice → [Gemini API] 오디오 분석
                                                          ↓
[Client] 초안 확인/수정 → [Server] /api/issue-invoice → [ASP API] → [국세청]
```

### 2.2 보조금 찾기 흐름

```
[Client] 사업장 정보 입력/음성 → [Server] /api/analyze-profile → [Gemini API]
                                                          ↓
[Client] "지원금 찾기" 클릭 → [Server] /api/find-subsidies → [Mock DB + Gemini RAG]
                                                          ↓
[Client] 카드 형태 결과 표시
```

### 2.3 핵심 설계 원칙
- **Security**: 인증서 비밀번호·API 키는 절대 클라이언트에 노출하지 않음
- **UI/UX**: AI 추출 데이터는 반드시 사용자 시각 확인·수정 단계 강제
- **Type Safety**: TypeScript 인터페이스로 프론트-백엔드 데이터 계약 명확화

## 7. 참고: Cursor 개발 진행 방식

1. `.cursorrules`에 마스터 프롬프트 설정
2. Step 1 ~ Step 7 프롬프트를 **순서대로** Cursor Composer(Cmd+I)에 입력
3. 에러 발생 시 에러 메시지를 Cursor 채팅창에 붙여넣어 수정 요청
4. Popbill/Barobill 회원가입 후 API Key·인증서를 `.env`에 설정하고 Mock 코드를 실제 코드로 교체

## 2. 확인 필요 (사용자 환경에 따라)

| 구분 | 항목 | 확인 방법 | 미완료 시 |
|------|------|-----------|-----------|
| **환경변수** | `.env.local`에 POPBILL_LINKID, POPBILL_SECRET_KEY | 설정 페이지에 노란 경고 없음 | 사업자 등록 폼 미표시 |
| **환경변수** | GEMINI_API_KEY | 음성/텍스트 분석 동작 | 분석 실패 |
| **환경변수** | AUTH_SECRET | 로그인 유지 | 세션 불안정 |
| **사용자** | 회원가입 완료 | `/login`에서 가입 | 설정 접근 불가 |
| **사용자** | 사업자 등록 완료 | `/settings`에서 폼 제출 | 실제 발행 불가 |
| **사용자** | 인증서 등록 완료 | "인증서 등록 창 열기" 후 업로드 | 실제 발행 불가 |
| **팝빌** | 포인트 충전 | 팝빌 사이트에서 확인 | 발행 시 포인트 부족 오류 |

## 3. 남은 할 일 (선택/향후)

| 구분 | 항목 | 우선순위 | 설명 |
|------|------|----------|------|
| **테스트** | 실제 발행 테스트 | 높음 | 팝빌 테스트 환경에서 1건 발행 후 결과 확인 |
| **테스트** | 국세청 전송 결과 확인 | 높음 | GetInfo 또는 Webhook으로 전송 성공 여부 확인 |
| **운영** | POPBILL_IS_TEST=false 전환 | 중간 | 실제 국세청으로 전송 (운영 전환 시) |
| **운영** | 팝빌 운영 포인트 충전 | 중간 | 실제 발행 시 포인트 과금 |
| **기능** | 수정세금계산서 발행 | 낮음 | 오류 시 수정 발행 (필요 시 구현) |
| **기능** | 발행취소 (CancelIssue) | 낮음 | 익일 전송 시에만 가능, UI 추가 |
| **데이터** | 지원금 실제 API 연동 | 낮음 | 공공데이터 기업마당 API (현재 Mock) |
| **보안** | AUTH_SECRET 강화 | 낮음 | 운영 시 `openssl rand -base64 32`로 생성 |

## 4. 실제 발행 가능 여부 (체크리스트)

아래가 **모두** 완료되면 실제 국세청 발행이 가능합니다.

| # | 항목 | 완료 |
|---|------|:----:|
| 1 | `.env.local`에 POPBILL_LINKID, POPBILL_SECRET_KEY 설정 | ☐ |
| 2 | 회원가입 후 로그인 | ☐ |
| 3 | 설정에서 사업자 등록 완료 | ☐ |
| 4 | 설정에서 인증서 등록 완료 | ☐ |
| 5 | 팝빌 포인트 충전 (테스트 환경) | ☐ |
| 6 | 메인에서 공급받는자 사업자번호 10자리 입력 | ☐ |

## 5. 흐름 요약

```
[완료] 회원가입 → 로그인 → 사업자 등록 → 인증서 등록
         ↓
[완료] 음성/텍스트 입력 → Gemini 분석 → 초안 확인
         ↓
[완료] 발행 → RegistIssue → 팝빌 → 국세청 (발행 즉시 전송)
```

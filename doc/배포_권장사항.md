# 배포 권장사항: Vercel 사용

## Cloudflare Pages 배포가 실패하는 이유

이 프로젝트는 **Node.js 전용 패키지**를 사용합니다:

| 패키지 | Node.js API | Edge Runtime |
|--------|-------------|--------------|
| **popbill** | crypto, https, http, fs, zlib | ❌ 미지원 |
| **@/lib/auth** | fs, bcrypt | ❌ 미지원 |

**@cloudflare/next-on-pages**는 **Edge Runtime만** 지원합니다.  
Edge Runtime에서는 `crypto`, `https`, `fs` 등 Node.js 내장 모듈을 사용할 수 없습니다.

→ **Cloudflare Pages로는 이 프로젝트를 배포할 수 없습니다.**

---

## 권장: Vercel 배포

**Vercel**은 Next.js를 기본 지원하며, Node.js API를 그대로 사용할 수 있습니다.

### Vercel 배포 절차

1. https://vercel.com 접속 → GitHub 로그인
2. **Add New Project** → `heosanghun/voice-invoice` 저장소 선택
3. **Framework Preset**: Next.js (자동 감지)
4. **Environment Variables** 설정:
   - `GEMINI_API_KEY`
   - `AUTH_SECRET`
   - `POPBILL_LINKID`
   - `POPBILL_SECRET_KEY`
   - `POPBILL_IS_TEST` (true/false)
5. **Deploy** 클릭

### 참고

- Vercel은 `npm run build`를 자동으로 실행합니다.
- 별도 어댑터나 Edge Runtime 설정이 필요 없습니다.
- `data/users.json`은 Vercel 서버리스 환경에서도 **쓰기 불가**이므로, 운영 시 DB 전환이 필요합니다.

# 국세청 세금계산서 실제 발행 연동 구현 가이드

> **현재 상태**: Mock 발행 (MOCK-xxx 승인번호)  
> **목표**: 실제 국세청 전송 및 공인인증서 기반 발행

---

## 1. 왜 국세청에 직접 연결할 수 없나요?

### 1.1 국세청(홈택스) 직접 연동의 한계

사용자가 말씀하신 **"국세청 로그인 → 사업자전환 → 공인인증서 로그인"** 흐름은 **홈택스 웹사이트**에서 사람이 직접 하는 절차입니다.

| 구분 | 설명 |
|------|------|
| **홈택스** | 국세청이 제공하는 웹 포털 (https://www.hometax.go.kr) |
| **API 제공** | ❌ **일반 개발자/기업에게 공개 API가 없음** |
| **접근 방식** | 브라우저에서 수동 로그인 후 화면 조작만 가능 |

즉, **우리 앱이 국세청 서버에 직접 HTTP API를 호출하는 것은 불가능**합니다. 국세청은 보안·감사·책임 소재를 위해 **공인된 ASP(Application Service Provider)만** 국세청 전송 권한을 부여합니다.

### 1.2 올바른 구조: ASP를 통한 간접 연동

```
[우리 앱] → [팝빌/바로빌 ASP API] → [국세청]
                ↑
         공인인증서 등록 (사전 1회)
```

- **ASP**: 국세청과 공식 계약을 맺은 전자세금계산서 대행 업체
- **대표 ASP**: 팝빌(Popbill), 바로빌(Barobill), 스마트빌 등
- **역할**: 우리가 ASP API로 세금계산서 데이터를 보내면, ASP가 **등록된 공인인증서로 전자서명** 후 국세청에 전송

## 2. 전체 흐름 (실제 발행까지)

### 2.1 사전 준비 (1회)

**이 앱에서의 흐름 (로컬 서버 기준):**

| 순서 | 작업 | 설명 |
|------|------|------|
| 1 | **회원가입** | `/login`에서 이메일·비밀번호로 가입 |
| 2 | **로그인** | 가입 후 로그인 |
| 3 | **사업자 등록** | `/settings`에서 사업자정보 입력 → 팝빌 JoinMember API로 연동회원 가입 |
| 4 | **공인인증서 등록** | `/settings`에서 "인증서 등록 창 열기" → 팝빌 팝업에서 인증서 업로드 |
| 5 | **실제 발행** | 메인에서 음성/텍스트 입력 → 공급받는자 사업자번호 입력 → 발행 |

**서버 측 사전 설정 (.env.local):**

| 변수 | 설명 |
|------|------|
| POPBILL_LINKID | 팝빌 파트너 링크아이디 (연동신청 후 발급) |
| POPBILL_SECRET_KEY | 팝빌 비밀키 |
| AUTH_SECRET | JWT 시크릿 (임의 문자열) |

### 2.2 공인인증서 등록 (가장 중요)

**공인인증서는 우리 앱에서 직접 다루지 않습니다.**  
팝빌이 제공하는 **인증서 등록 페이지**에서 사용자가 직접 등록합니다.

#### 방법 A: 팝빌 사이트에서 등록

1. https://www.popbill.com 로그인
2. **인증서 관리** 메뉴
3. **공동인증서 등록** 클릭
4. 인증서 파일(.pfx 또는 .p12) 선택 + 비밀번호 입력
5. 팝빌 인증서버에 암호화 저장

#### 방법 B: 팝빌 인증센터 (전자세금용 특수목적 인증서)

- https://cert.popbill.com
- **전자세금용 특수목적 인증서** 발급 (1년 2만원, 3년 4만원)
- 발급 후 팝빌에 자동 연동

#### 인증서가 필요한 이유

- 국세청 전송 시 **전자서명** 필수
- 서명 = “이 공급자가 이 세금계산서를 발행했다”는 법적 증명
- 공인인증서는 이 서명을 생성하는 수단

## 3. 기술적 구현 (팝빌 Node.js SDK)

### 3.1 패키지 설치

```bash
npm install popbill
```

### 3.2 환경변수 (.env.local)

```env
# 팝빌 API (필수)
POPBILL_LINKID=발급받은링크아이디
POPBILL_SECRET_KEY=발급받은비밀키
POPBILL_CORP_NUM=1234567890          # 사업자번호 (하이픈 제외)

# 테스트/운영 전환
POPBILL_IS_TEST=true                  # true=테스트, false=운영(실제 국세청)
```

### 3.3 발행 흐름 (RegistIssue)

```
1. 클라이언트: "이대로 세금계산서 발행하기" 클릭
2. 서버: InvoiceData 수신
3. 서버: 팝빌 Taxinvoice 객체로 변환
4. 서버: taxinvoiceService.registIssue() 호출
5. 팝빌: 등록된 공인인증서로 전자서명
6. 팝빌: 국세청 전송
7. 팝빌: ntsConfirmNum(국세청승인번호) 반환
8. 서버: ntsConfirmNum을 클라이언트에 전달
```

### 3.4 Taxinvoice 객체 필수 필드

| 필드 | 설명 | 예시 |
|------|------|------|
| writeDate | 작성일자 | 20240520 |
| issueType | 발행형태 | SELL(정발행) |
| taxType | 과세형태 | TAX(과세) |
| supplyCostTotal | 공급가액 합계 | 1000000 |
| taxTotal | 세액 합계 | 100000 |
| totalAmount | 합계금액 | 1100000 |
| invoicerCorpNum | 공급자 사업자번호 | 1234567890 |
| invoicerMgtKey | 공급자 문서번호 | INV-2024-001 |
| invoicerCorpName | 공급자 상호 | (주)우리회사 |
| invoiceeCorpNum | 공급받는자 사업자번호 | 9876543210 |
| invoiceeCorpName | 공급받는자 상호 | 동아 |
| invoiceeType | 공급받는자 유형 | CORPORATE(사업자) |
| itemList | 품목 목록 | [{itemName, qty, supplyCost, tax, ...}] |

### 3.5 사업자번호 조회

**"동아"만 입력했을 때** → 사업자번호가 필요합니다.

- **방법 1**: 사용자가 직접 사업자번호 입력 (폼에 필드 추가)
- **방법 2**: 팝빌 **기업정보조회 API**로 상호명 검색 → 사업자번호 조회
- **방법 3**: Gemini에 "사업자번호도 함께 추출해"라고 프롬프트 (정확도 낮음)

권장: **방법 1** – 폼에 공급받는자 사업자번호 입력란 추가

## 4. 구현 단계별 체크리스트

### Phase 1: 팝빌 사전 준비

- [ ] 팝빌 회원가입 (https://www.popbill.com)
- [ ] 사업자등록
- [ ] 공인인증서 등록 (팝빌 인증서버)
- [ ] API 연동신청 → LinkID, SecretKey 발급
- [ ] 테스트 환경 포인트 충전

### Phase 2: 코드 연동

- [ ] `npm install popbill`
- [ ] `.env.local`에 POPBILL_* 설정
- [ ] `issue-invoice` API에서 Mock 제거, 팝빌 `registIssue` 호출
- [ ] InvoiceData에 `buyerCorpNum`(공급받는자 사업자번호) 필드 추가
- [ ] 공급자 정보(상호, 주소 등)는 팝빌 사이트에 등록 또는 API로 설정

### Phase 3: 테스트

- [ ] `POPBILL_IS_TEST=true`로 테스트 환경 발행
- [ ] 승인번호 `ntsConfirmNum` 확인
- [ ] 팝빌 회원페이지 → 전자세금계산서 발행내역 확인

### Phase 4: 운영 전환

- [ ] `POPBILL_IS_TEST=false`로 변경
- [ ] 운영 포인트 충전
- [ ] 실제 국세청 전송 확인

## 5. 요약

| 질문 | 답변 |
|------|------|
| 국세청에 직접 연결할 수 있나요? | ❌ 불가. 공개 API 없음. |
| 공인인증서 로그인은 어떻게? | 팝빌 사이트/인증센터에서 **1회 등록** 후, 팝빌이 대신 서명 |
| "동아" 조회는? | 사업자번호 입력란 추가 또는 기업정보조회 API 활용 |
| 실제 발행은? | 팝빌 API `registIssue` 호출 → 팝빌이 국세청 전송 → `ntsConfirmNum` 반환 |
